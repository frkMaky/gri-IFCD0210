Cadenas de JavaScript

Las cadenas son para almacenar texto.

Las cadenas están escritas entre comillas.

Usando comillas

Una cadena de JavaScript tiene cero o más caracteres escritos entre comillas.

            let text = "John Doe";

Puede utilizar comillas simples o dobles:

            let carName1 = "Volvo XC60"; // Double quotes
            let carName2 = 'Volvo XC60'; // Single quotes

Nota Las cadenas creadas con comillas simples o dobles funcionan igual.

No hay diferencia entre los dos.

Comillas dentro de comillas

Puedes usar comillas dentro de una cadena, siempre y cuando no coincidan con las comillas que rodean la cadena:

            let answer1 = "It's alright";
            let answer2 = "He is called 'Johnny'";
            let answer3 = 'He is called "Johnny"';

Cadenas de plantilla

Las plantillas se introdujeron con ES6 (JavaScript 2016).

Las plantillas son cadenas entre comillas invertidas (`This is a template string` ).

Las plantillas permiten comillas simples y dobles dentro de una cadena:

            let text = `He's often called "Johnny"`;

Nota Las plantillas no son compatibles con Internet Explorer.

Longitud de la cadena

Para encontrar la longitud de una cadena, use la propiedad incorporada length:

            let text = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";
            let length = text.length;

Escapar caracteres

Debido a que las cadenas deben escribirse entre comillas, JavaScript malinterpretará esta cadena:

            let text = "We are the so-called "Vikings" from the north.";

Para resolver este problema, puede utilizar un carácter de escape de barra invertida .

El carácter de escape de barra invertida ( \) convierte los caracteres especiales en caracteres de cadena:

CodigoResultadoDescripción
\''Comilla simple
\""Comilla doble
\\\Barra invertida
            \" inserta una comilla doble en una cadena:
            
            let text = "We are the so-called \"Vikings\" from the north.";
        
            \' inserta una comilla simple en una cadena:
            
            let text= 'It\'s alright.';
        
            \\ inserta una barra invertida en una cadena:
            
            let text = "The character \\ is called backslash.";

Otras seis secuencias de escape son válidas en JavaScript:

CodigoResultado
\bBackspace
\fForm Feed
\nNew Line
\rCarriage Return
\tHorizontal Tabulator
\vVertical Tabulator

NotaLos 6 caracteres de escape anteriores fueron diseñados originalmente para controlar máquinas de escribir, teletipos y máquinas de fax. No tienen ningún sentido en HTML.

Rompiendo líneas largas

Para facilitar la lectura, a los programadores les gusta evitar líneas de código largas.

Una forma segura de dividir una declaración es después de un operador:

            document.getElementById("demo").innerHTML =
            "Hello Dolly!";

Una forma segura de dividir una cadena es mediante la suma de cadenas:

            document.getElementById("demo").innerHTML = "Hello " +
            "Dolly!";

Cadenas de plantilla

Las plantillas se introdujeron con ES6 (JavaScript 2016).

Las plantillas son cadenas entre comillas invertidas ("Esta es una cadena de plantilla").

Las plantillas permiten cadenas multilínea:

            let text =
            `The quick
            brown fox
            jumps over
            the lazy dog`;

Nota Las plantillas no son compatibles con Internet Explorer.

Cadenas de JavaScript como objetos

Normalmente, las cadenas de JavaScript son valores primitivos, creados a partir de literales:

            let x = "John";

Pero las cadenas también se pueden definir como objetos con la palabra clave new:

            let y = new String("John");
        
            let x = "John";
            let y = new String("John");

No cree objetos Strings.

La palabra clave new complica el código y ralentiza la velocidad de ejecución.

Los objetos String pueden producir resultados inesperados:

Cuando se utiliza el operador ==, x e y son iguales :

            let x = "John";
            let y = new String("John");

Cuando se utiliza el operador ===, x e y no son iguales :

            let x = "John";
            let y = new String("John");

Note la diferencia entre (x==y)y (x===y).

(x == y)¿verdadero o falso?

            let x = new String("John");
            let y = new String("John");

(x === y)¿verdadero o falso?

            let x = new String("John");
            let y = new String("John");

La comparación de dos objetos JavaScript siempre devuelve false .
