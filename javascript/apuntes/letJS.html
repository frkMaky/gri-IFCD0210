JavaScript let

La palabra clave let se introdujo en ES6 (2015)

Las variables declaradas con let tienen alcance de bloque

Las variables declaradas con let deben declararse antes de su uso.

Las variables declaradas con let no se pueden volver a declarar en el mismo ámbito

Alcance del bloque

Antes de ES6 (2015), JavaScript no tenía Block Scope .

JavaScript tenía alcance global y alcance de función .

ES6 introdujo las dos nuevas palabras clave de JavaScript: let y const.

Estas dos palabras clave proporcionaron Block Scope en JavaScript:

            No se puede acceder a las variables declaradas dentro de un bloque { } desde fuera del bloque:
            {
            let x = 2;
            }
            // x can NOT be used here

Alcance global (global scope)

Las variables declaradas con var siempre tienen Global Scope .

Las variables declaradas con la palabra clave var NO pueden tener alcance de bloque:

            Variables declaradas con var dentro de un bloque { } pueden accederse desde fuera del bloque:
            {
            var x = 2;
            }
            // x CAN be used here

No se puede volver a declarar

Las variables definidas con let no se pueden volver a declarar.

No puedes redeclarar accidentalmente una variable declarada con let.

Con let no puedes hacer esto:

            let x = "John Doe";
            let x = 0;

Las variables definidas con var se pueden redeclarar.

Con var puedes hacer esto:

            var x = "John Doe";
            var x = 0;

Redeclaración de variables

Redeclarar una variable usando la palabra clave var puede plantear problemas.

Redeclarar una variable dentro de un bloque también redeclarará la variable fuera del bloque:

            var x = 10;
            // Here x is 10
            {
            var x = 2;
            // Here x is 2
            }
            // Here x is 2

Redeclarar una variable usando la palabra clave let puede resolver este problema.

Redeclarar una variable dentro de un bloque no redeclarará la variable fuera del bloque:

            let x = 10;
            // Here x is 10
            {
            let x = 2;
            // Here x is 2
            }
            // Here x is 10

Diferencia entre var, let y const

Alcance Redeclarar Reasignar
var NO SI SI
let SI NO SI
const SI NO NO

¿Lo que es bueno?

Let y const tiene alcance de bloque .

Let y const no se puede volver a declarar .

Let y const debe declararse antes de su uso.

¿Qué no es bueno?

varno es necesario declararlo.

Redeclaración

Se permite redeclarar una variable JavaScript con varen cualquier parte de un programa:

            var x = 2;
            // Now x is 2
            var x = 3;
            // Now x is 3

Con let, NO se permite redeclarar una variable en el mismo bloque:

Ejemplo

            var x = 2; // Allowed
            let x = 3; // Not allowed
            {
            let x = 2; // Allowed
            let x = 3; // Not allowed
            }
            {
            let x = 2; // Allowed
            var x = 3; // Not allowed
            }

Redeclarar una variable con let, en otro bloque, ESTÁ permitido:

            let x = 2; // Allowed
            {
            let x = 3; // Allowed
            }
            {
            let x = 4; // Allowed
            }

Let puede ser usada antes de ser declarada

Puede utilizar la variable antes de declararla:

            Esto esta bien:
            carName = "Volvo";
            var carName;

El uso de una variable let antes de declararla dará como resultado ReferenceError:

            carName = "Saab";
            let carName = "Volvo";
